# Next Project Fast Start Checklist

Internal reusable checklist.

## Before Build

- Confirm final domain and preview domain.
- Confirm active vertical, market, services, and safety rules.
- Confirm DNS owner and hosting target.
- Create `.env.example` with placeholders only.
- Create manual owner steps for DNS, search console, analytics, and phone tracking.

## Build

- Create homepage with public-safe positioning.
- Create service, city, city-service, problem, cost, blog, FAQ, contact, partner, privacy, terms, and disclosure routes.
- Add schema utilities, breadcrumbs, internal links, sitemap, and robots.
- Add form placeholder with no secrets.
- Add original or licensed visuals and track permissions.

## QA

- Run forbidden public-term scan.
- Run build, typecheck, and QA scripts.
- Check sitemap URL count.
- Check robots response.
- Check mobile CTA and form.
- Update reports before commit.

## Deployment

- Commit and push.
- Let hosting deploy from GitHub.
- Check production home, robots, sitemap, and key index files.
- Submit or request indexing only when owner-approved.
