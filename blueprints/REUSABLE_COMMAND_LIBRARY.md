# Reusable Command Library

Internal command library. Adapt values before reuse.

## Foundation Command

Build the Next.js local service request foundation for one vertical and one market. Use safe public copy, no secrets, no fake claims, sitemap and robots, schema utilities, request form placeholder, manual owner steps, and QA reports.

## Content Scale Command

Create service, city, city-service, problem, cost guide, blog, FAQ, and internal link pages using structured data files. Keep all claims verifiable and avoid copied competitor content or images.

## Public Cleanup Command

Scan public code and assets for internal business model terms. Remove public references to prompts, internal strategy, automation, routing economics, private tools, or reusable platform strategy. Preserve internal terms only in private docs.

## Indexing Command

Only after owner approval, verify production domain, sitemap, robots, Search Console status, Bing or IndexNow status, and submit URLs through approved channels. Do not rerun submissions unnecessarily.

## Day 3 Design Command

Upgrade homepage, global styles, CTA system, forms, disclosures, safe visuals, public copy, reports, reusable blueprints, and obsolete softcopy cleanup. Build, QA, commit, push, then verify live URLs.
