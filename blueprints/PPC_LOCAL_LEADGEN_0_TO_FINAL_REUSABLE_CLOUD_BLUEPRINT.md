# PPC Local Leadgen 0 To Final Reusable Cloud Blueprint

Internal blueprint. Do not publish this file as public website copy.

## Purpose

Build a reusable local lead-generation cloud project that can launch one home-service vertical at a time with safe SEO, AEO, compliance, provider handling, and QA.

## Core Phases

1. Integration setup: GitHub, Codex, local PowerShell, Colab support files, D-drive backup, and manual owner steps.
2. Foundation: Next.js structure, domain-safe environment variables, sitemap, robots, schema utilities, pages, forms, and tracking placeholders.
3. Content scale: service pages, city pages, city-service pages, problem pages, cost guides, blogs, FAQs, internal links, and content QA.
4. Public cleanup: remove internal strategy from public pages, preserve required disclosures, and keep claims verifiable.
5. Deployment: Vercel production path, DNS owner steps, HTTPS verification, GSC, sitemap, IndexNow, and live QA.
6. Growth: analytics, conversion tracking, provider onboarding, approved citations, content expansion, and outreach only after owner approval.

## Reusable Rules

- One active vertical and market at launch.
- No fake local office claims.
- No fake reviews, addresses, licenses, insurance, or badges.
- No copied competitor images.
- No paid ads unless the owner separately approves an ads project.
- No secrets in repo files.

## Required Outputs

- Build-passing website.
- Public-safe copy.
- Sitemap and robots.
- Reports for indexing, content QA, schema, design, and launch.
- Manual owner steps.
- Backup-ready blueprint files.
