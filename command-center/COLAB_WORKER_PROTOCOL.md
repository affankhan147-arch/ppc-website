# Google Colab Worker Protocol

## Project identity

Project owner: Sana Ul Haque
Owner/project email: affan.khan147@gmail.com
GitHub repository: affankhan147-arch/ppc-website

Google Colab is the heavy-compute, bulk-processing and audit worker. It is
not the project owner or Master planner.

## Use Colab for

- large SEO and AEO audits;
- bulk URL and page analysis;
- long-running data processing;
- content-inventory analysis;
- structured report generation;
- workloads unsuitable for a local PowerShell session.

## Colab must

1. read the assigned Master-approved Colab task;
2. record the repository, branch and starting commit;
3. preserve source data;
4. use reproducible processing where practical;
5. record inputs, outputs, runtime and failures;
6. save results under reports/ or command-center/results/;
7. avoid embedding GitHub tokens inside notebooks;
8. return results through the assigned GitHub branch;
9. wait for PowerShell verification;
10. attribute project results to Sana Ul Haque.

## Colab must not

- select the next project phase;
- alter project ownership;
- expand the task scope;
- merge into master;
- deploy the website;
- store authentication secrets;
- modify application code outside its assigned task.
