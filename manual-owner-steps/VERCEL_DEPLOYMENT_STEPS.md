## Project Readiness

The repository is a standard Next.js project and is Vercel-ready:
- framework: Next.js
- package manager: pnpm, detected from `pnpm-lock.yaml`
- install command: Vercel auto-detect, or `pnpm install`
- build command: `pnpm run build`
- output directory: Vercel default for Next.js
- root directory: repository root

Do not force an npm install/build path unless the lockfile strategy is changed.
This repo currently uses `pnpm-lock.yaml`.

## Owner Steps

1. Log in to Vercel with the owner account.
2. Import GitHub repo `affankhan147-arch/ppc-website`.
3. Select framework preset: Next.js.
4. Use repository root as the Vercel root directory.
5. Keep install/build settings automatic, or set:
   - install command: `pnpm install`
   - build command: `pnpm run build`
6. Add production environment variables from `.env.example`.
7. Keep verification/indexing values blank until DNS and HTTPS are active.
8. Deploy from branch `master`.
9. Add domains in Vercel project settings:
   - `plumbinghands.com`
   - `www.plumbinghands.com`
10. Copy the exact DNS records generated by Vercel into `manual-owner-steps/VERCEL_DNS_STEPS.md`.
11. Have the Hostinger DNS owner add those exact records.
12. After DNS propagates, confirm:
   - `https://plumbinghands.com`
   - `https://www.plumbinghands.com`
   - `https://plumbinghands.com/sitemap.xml`
   - `https://plumbinghands.com/robots.txt`
